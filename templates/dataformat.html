File Data Structure Reference

Critical Rule: The first row must always be headers (for CSV & Excel).
Missing values in data rows are allowed, but headers must exist, and they must not be merged or missing.
If a value is missing, then sending mail to that email address will be skipped.

XLSX / Excel Format

Correct Structure (Header in First Row)

id name email department salary
1John Doejohn.doe@example.comSales55000
2Jane Smithjane.smith@example.comMarketing60000
3Michael Brownmichael.brown@example.comIT75000
4Emily Davisemily.davis@example.comHR52000
5David Wilsondavid.wilson@example.comFinance68000
6Sarah Johnsonsarah.johnson@example.comIT72000
7Daniel Leedaniel.lee@example.comOperations58000
8Laura Martinezlaura.martinez@example.comSales56000
9James Taylorjames.taylor@example.comMarketing61000
10olivia.anderson@example.comHR54000

The header row is present as the first row.

Each column has a unique header.

There are no merged cells in the headers.

Data follows immediately after the headers.

Empty cells in data rows are allowed.

Wrong: Merged Header Cells

Employee Data
1John Doejohn.doe@example.comSales55000
2Jane Smithjane.smith@example.comMarketing60000

Merged cells prevent proper column identification.

Each column must have its own header.

Wrong: Missing Header Row

1John Doejohn.doe@example.comSales55000
2Jane Smithjane.smith@example.comMarketing60000

The system might detect column names, but selecting emails will be difficult.

The file may be processed incorrectly.

CSV Format

Correct CSV Structure

data.csv - Notepad
id,name,email,department,salary 1,John Doe,john.doe@example.com,Sales,55000 2,Jane Smith,jane.smith@example.com,Marketing,60000 3,Michael Brown,michael.brown@example.com,IT,75000 4,Emily Davis,emily.davis@example.com,HR,52000 5,David Wilson,david.wilson@example.com,Finance,68000 6,Sarah Johnson,sarah.johnson@example.com,IT,72000 7,Daniel Lee,daniel.lee@example.com,Operations,58000 8,Laura Martinez,laura.martinez@example.com,Sales,56000 9,James Taylor,james.taylor@example.com,Marketing,61000 10,Olivia Anderson,olivia.anderson@example.com,HR,54000

The first line contains the headers.

Clean structure with no extra rows.

Correct: CSV with Empty Values

notepad.txt
col1,col2,col3,col4,col5 1,John Doe,john.doe@example.com,Sales,55000 2,Jane Smith,,Marketing,60000 3,Michael Brown,michael.brown@example.com,,

Header is the first row.

Empty cells in data rows are allowed but will be skipped when sending email.

Columns are not merged.

Wrong: Missing Header Row

data_error.csv
1,John Doe,john.doe@example.com,Sales,55000 2,Jane Smith,jane.smith@example.com,Marketing,60000 3,Michael Brown,michael.brown@example.com,IT,75000

The system might detect column names, but selecting emails will be difficult.

The first data row may be incorrectly treated as the header.

Wrong: Extra Content Before Header

report.csv
Employee Salary File Generated: 2024 id,name,email,department,salary 1,John Doe,john.doe@example.com,Sales,55000

Extra rows confuse the parser.

The system cannot identify the true header row.

Wrong: Multiple Header Rows

details.csv
Employee Details,,,,, ID,Name,Email,Department,Salary 1,John Doe,john.doe@example.com,Sales,55000 2,Jane Smith,jane.smith@example.com,Marketing,60000

The system cannot detect which row contains the actual headers.

The file will be rejected.

Wrong: Empty or Partial Headers

partial.csv
id,name,,department, 1,John Doe,john.doe@example.com,Sales,55000 2,Jane Smith,jane.smith@example.com,Marketing,60000

Missing column headers create ambiguity.

The system may auto-generate generic names or fail.

JSON Format

Correct JSON Structure (Array of Objects)

[ { "id": 1, "name": "John Doe", "email": "john.doe@example.com", "department": "Sales", "salary": 55000 }, { "id": 2, "name": "Jane Smith", "email": "jane.smith@example.com", "department": "Marketing", "salary": 60000 }, { "id": 3, "name": "Michael Brown", "email": "michael.brown@example.com", "department": "IT", "salary": 75000 }, { "id": 4, "name": "Emily Davis", "email": "emily.davis@example.com", "department": "HR", "salary": 52000 }, { "id": 5, "name": "David Wilson", "email": "david.wilson@example.com", "department": "Finance", "salary": 68000 }, { "id": 6, "name": "Sarah Johnson", "email": "sarah.johnson@example.com", "department": "IT", "salary": 72000 }, { "id": 7, "name": "Daniel Lee", "email": "daniel.lee@example.com", "department": "Operations", "salary": 58000 }, { "id": 8, "name": "Laura Martinez", "email": "laura.martinez@example.com", "department": "Sales", "salary": 56000 }, { "id": 9, "name": "James Taylor", "email": "james.taylor@example.com", "department": "Marketing", "salary": 61000 }, { "id": 10, "name": "Olivia Anderson", "email": "olivia.anderson@example.com", "department": "HR", "salary": 54000 } ]

Array format for multiple records.

Consistent keys across all objects.

Proper JSON syntax.

Wrong: Inconsistent Structure

[ { "id": 1, "name": "John Doe" }, { "email": "jane.smith@example.com", "salary": 60000 } ]

Missing keys in different objects.

Inconsistent field structure.

Makes data processing difficult.

Best Practices Summary:
  • CSV/Excel: Always include a complete header row as the first row.
  • Headers: Use descriptive, unique column names without merging.
  • Empty Values: Empty data cells are fine, but empty header cells are not.
  • Clean Files: No title rows, metadata, or extra content before headers.
  • JSON: Use array format with consistent object structure.
  • Consistency: All records should have the same fields/columns.
